MacDown语法

发表于 2017-01-16

导语

Markdown 的目标是实现「易读易写」.

Markdown 的语法全由一些符号所组成，这些符号经过精挑细选，其作用一目了然。

Markdown 是一种书写的格式，HTML 是一种发布的格式。

Markdown 的格式语法只涵盖纯文本可以涵盖的范围。不在 Markdown 涵盖范围之内的标签，都可以直接在文档里面用 HTML 撰写。不需要额外标注这是 HTML 或是 Markdown；只要直接加标签就可以了。

Markdown 不是想要取代 HTML，甚至也没有要和它相近，它的语法种类很少，只对应 HTML 标记的一小部分。Markdown 的构想不是要使得 HTML 文档更容易书写。在我看来， HTML 已经很容易写了。Markdown 的理念是，能让文档更容易读、写和随意改。

1.4.1内链接

  1. 网址可用尖括号包裹使其可点击: http://macdown.uranusjr.com
    下面是网址示例代码,直接copy到macdown左侧中:

    下面是邮箱示例代码,直接copy到macdown左侧中:

  3. 也许你需要一些这样的链接文本: Macdown Website ,
    下面是链接文本示例代码,直接copy到macdown左侧中:

1.4.2 其他写法

有时链接太长,页面显得很混乱,你可以将所有链接统一放在一起定义,之后根据情况具体去调用.

  1. 链接定义:
    [arbitrary_id]: http://macdown.uranusjr.com
    [Baidu]: http://macdown.uranusjr.com
    [jd_id]: https://www.jd.com
  2. 可在文章任一地方使用已经定义的链接:
    一个链接 使用格式[][]:[一个链接][arbitrary_id]
    百度链接 使用格式[][]:[百度链接][Baidu]
    京东商城 使用格式[][]:[京东商城][jd_id]

1.5 图片

1.5.1 内链接

图片的引用:叹号+中括号+小括号 ![名称随意](图片url) 如:
![M12](http://macdown.uranusjr.com/static/base/img/logo-160.png)
![M34](http://ojupj7fr2.bkt.clouddn.com/blog_xintouxiang.png)

1.5.2 其他写法
  1. 先定义
    [image-id]: http://ojupj7fr2.bkt.clouddn.com/blog_touxiang.png
  2. 定义完,可在文章任意地方调用:
    ![Alt Image Text][image-id]

1.7 引用

尖括号 > 被用来块引用.
从技术上讲,引用好几段内容,并不是每一行都需要以尖括号>开始,只要段落与段落之间没有空行就行

1.8 代码

代码: 用单引号(键位在tab键上方)包裹
`private void initData(){}`
如果你的代码中有单引号并且需要展示,你可以使用两次单引号包裹已经存在的单引号:
``Code with `backticks` ``

1.9 代码块

显示一个代码块: 先空一行,使用tab键或者缩进四个空格,在写入代码,具体自己试验。 

print('This is a code block')
print('The block must be preceded by a blank line')
print('Then indent at least 4 spaces or 1 tab')
print('Nesting does nothing. Your code is displayed Literally')
